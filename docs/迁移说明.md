# MotionCore 迁移说明（变更记录 / Breaking Changes）

本文用于记录对业务侧/使用者可能产生影响的文档与约定变更。

## 2026-01：移除早期“控制面（RPC）/事件流（Pub/Sub）”设计建议文档

- 变更：移除一份早期“设计建议/背景材料”文档（历史材料，不再保留在仓库中）。
- 原因：该文档包含早期讨论与建议性内容，容易被误当作“当前开发规范”，与已落地的 `wxz::framework` 约定产生歧义。
- 替代入口（唯一推荐）：[框架层约定与用法.md](框架层约定与用法.md)

### 你需要做什么

- 如果的笔记/README/外部链接仍指向已删除文件：改为指向 [框架层约定与用法.md](框架层约定与用法.md)。
- 如果在实现上沿用了旧文档里的命名/Topic：以 framework 文档的默认约定为准，尤其是 RPC：
  - 请求：`/svc/<service>/rpc/request`
  - 响应：`/svc/<service>/rpc/reply`

### 不变的东西

- 业务侧 API 推荐以 `wxz::framework` 为入口（Node/Service/Client/Parameters/Timer/Rate/Lifecycle + 统一 QoS）。

## 2026-01：并发模型文档收敛

- 变更：移除单独的“并发与异步执行模型”文档。
- 原因：并发/线程模型的“可执行约定”已经收敛到 framework 文档，避免多入口造成歧义。
- 替代入口：见 [框架层约定与用法.md](框架层约定与用法.md) 的“排障速查/指标口径”。
